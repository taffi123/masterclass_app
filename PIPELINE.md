# CI/CD pipeline для Laravel

## Для чего нужен пайплайн

Этот пайплайн нужен для автоматической проверки проекта Laravel. Он запускается после каждого `push` или `pull request` в основные ветки:

- develop
- uat
- main

## Окружения

В проекте есть несколько файлов с настройками для разных сред:

- `.env.dev` — используется для разработки
- `.env.uat` — используется для тестирования приложения
- `.env.prod` — используется для рабочей версии приложения
- `.env.ci` — используется для запуска пайплайна

Обычный файл `.env` в репозиторий не загружается, потому что он находится в `.gitignore`.

## Как работает пайплайн

### 1. Запуск тестов

Сначала подготавливается Laravel-проект:

bash
cp .env.ci .env
php artisan key:generate --force
php artisan migrate --force
php artisan test --coverage --min=50

На этом этапе:

копируется файл настроек для CI
создаётся ключ приложения
запускаются миграции
запускаются тесты

Также проверяется покрытие кода тестами. Оно должно быть не меньше 50%.

Если тесты не проходят или покрытие меньше 50%, пайплайн остановится.

2. Проверка кода через PHPStan

Дальше запускается статический анализ:

./vendor/bin/phpstan analyse --error-format=github

Он ищет возможные ошибки в коде ещё до запуска приложения.

Если находятся ошибки, пайплайн завершается.

3. Проверка стиля кода

После этого запускается Laravel Pint:

./vendor/bin/pint --test

Он проверяет форматирование и стиль написания кода.

Если есть ошибки оформления, пайплайн не проходит.

4. Имитация деплоя

Если все предыдущие проверки успешно завершились, запускается этап деплоя.

Для разных веток используются разные файлы:

Ветка	Окружение
develop	.env.dev
uat	.env.uat
main	.env.prod

Для production настроено дополнительное подтверждение перед запуском.
